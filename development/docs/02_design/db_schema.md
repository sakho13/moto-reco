# DBスキーマ設計

## 基本

### 命名規則

- テーブル名は`UpperCamelCase`を使用
- カラム名は`lowerCamelCase`を使用
- プレフィックスは次のルールに従う
  - マスターテーブル: `M` (例: `MUser`, `MProduct`)
  - トランザクションテーブル: `T` (例: `TOrder`, `TPayment`)
  - ログテーブル: `L` (例: `LLogin`, `LTransaction`)

## テーブル設計

| テーブル名           | 説明                                                         |
| -------------------- | ------------------------------------------------------------ |
| MBike                | バイク情報を格納するマスターテーブル(同じシリーズでも年式別) |
| MMaintenanceType     | メンテナンス種類情報を格納するマスターテーブル               |
| MAuthProvider        | 認証情報を格納するマスターテーブル                           |
| MUser                | ユーザー情報を格納するマスターテーブル                       |
| TUserBike            | ユーザが所有しているバイクの情報を格納するテーブル           |
| TUserBikeMaintenance | ユーザのバイクのメンテナンス履歴情報を格納するテーブル       |
| TUserBikeFuel        | ユーザのバイクの燃料履歴情報を格納するテーブル               |
| TUserBikeLiability   | ユーザのバイクの自賠責履歴情報を格納するテーブル             |
| TUserBikeInsurance   | ユーザのバイクの任意保険履歴情報を格納するテーブル           |
| LUserSyncLog         | 同期実施情報を格納するログテーブル                           |
